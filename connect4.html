Connect 4

Tasked to convert a classic board game into an online multiplayer experience I chose Connect 4. I used Node.js and Socket.IO for real time communication and their Rooms feature. Consequently, I didn't have to use a different language to write the backend and stuck with HTML, CSS and Javascript to write the entire thing.

Github Repository

Context

For my Web Development class we were pushed to learn Node.js for backend development. In conjunction with Node.js, we learned Socket.IO. Socket.IO was excellent as it is the library that is used for all .IO games for its bi-directional communication in real time and has many additional feature that bare bones web sockets don't like Rooms. We had many requirements for this project; assign new players a name, remember it through cookies, allow players to create their own matches (rooms), allow players to join existing rooms via a room code or joining a random open room, and giving players the ability to choose different colour themes. All of these were met and the game can be downloaded and tested via the Github repository. The node dependencies need to be downloaded before you can test it locally.
