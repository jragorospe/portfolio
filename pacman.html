Pacman RTS Demo

Been playing around with Unity for the past couple months and decided to tackle something a bit more difficult for my second "complete" prototype. The Real Time Strategy (RTS) genre has been less popular in this generation of games, not only for players but developers as well. It is considered to be one of the more difficult genres to develop for since the programming is a lot more math and algorithm heavy (unit types, pathfinding, AI). I wanted to program the basic camera controls, unit behaviours and basic enemy types in this personal project and with the suite of tools available in Unity, I was able to create a small mini-game.

About

Every good game starts with solid planning and a design document. I needed to identify what was essential in a RTS game. Since I have not played too many RTS games myself I needed to see what the industry standards were through the genre's most popular titles: Starcraft 2, Warcraft 3, Command and Conquer, Sim City, etc. So I created a list of needs to secure the feel of an RTS game and built my game idea on top of the essentials...

Orbital World Camera - RTSs live and die by their camera. We want a camera that you can control with the keyboard and mouse that rotates around the center of focus. We also need to make sure that when units and objects are selected, the player can choose to snap the camera to focus and follow when necessary.

Click/Box Selector - this is going to be the most difficult thing to do. RTS have single click and box selection options. Single click is easy but box selection, depending on how you want it done, is difficult. In a 3D game we can either create a box that selects anything in its area. This is janky since we are looking on an angle and the world is usually at a 45 degree rotation. What we really want is the ability to drag on the screen and have a box be drawn in the UI rather than spawn GameObject in 3D space. This will be done using a RayCast technique and a lot of math. By using Rays we can find the 4 points that indicate the corners of the box we create and then create the geometry necessary to grab selectable units. The points then create a box and by identifying the triangles in the box we can create a mesh which Unity recognizes as a shape. We want to be able to select more than one thing so we keep the selected units in a dictionary, adding and removing when necessary. Deselecting will be handled by clicking on a layer that is not selectable (ie. Ground).

Pathfinding - this one may be difficult depending on what Unity has to offer. To start we will use Unity’s NavMesh tool to set which surfaces the player can travel on. Then we will implement a pathfinding algorithm (Dykstra’s, FlowField, etc.) to have units optimally move through the environment.

Unit/Selection Indicator - when a unit is selected there must be a clear indication that it is. We also need to differentiate when we select a friendly or enemy unit. We will use green and red rings similar to Warcraft 3. Other colors of rings can be added to indicate different additional states.

After identifying the essentials I got to work implementing these things and building on top of the main tools. I added a basic enemy turret, main menu and pause menu. I used a lot of Unity's built in tools to create various things for the game: particle system for confetti effect, ProBuilder for the Pacman Maze, NavMesh for pathfinding, and TextMeshPro for UI elements.
