Peg Solitaire Solver

This was a project made for my Programming Paradigms class where we were learning how to use functional programming languages (Haskell and Prolog). Functional programming is a programming paradigm where programs are constructed by applying and composing functions. Each function is treated like a value, it returns something back and you cannot store mutable data in them. This style of programming allows for very transparent, intentional outputs, storing only what is necessary.

Prolog is a logic based programming language, known for its role in AI. Prolog is intended primarily as a declarative programming language meaning its logic is expressed as a set of Facts and Rules. Prolog is fantastic when you know where to start and where to end but don't know how to bridge the gap. In the case of Peg Solitaire we know the start board and the final position but we don't know the moves to get there.



In this program, after setting up the Rules and the Facts, we use brute force to find out all the possible sets of moves that can lead to a completed board. This, however is not enough as it would take forever given the sheer number of possible moves that can be taken. We take it a step further and use independent checks to make sure we aren't backtracking. How this is done is we keep a log of moves already made and make sure we don't take moves in the log. Once we find an overlap (matching move), we terminate the attempt and go to the next one. This alone cuts the process significantly, solving the main board in minutes. Next we use Pagoda Predicates. This is adding "weights" to certain positions on the board that are more favourable/unfavourable to take. When the program decides to make a jump (move) it will avoid negative weights and favour higher valued positions. With this we can solve the full board in 3 seconds.
